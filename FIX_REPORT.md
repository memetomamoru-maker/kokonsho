# 古今抄 v19 UIUX最小修正レポート

## 正として扱うファイル

`kokonsho_github_ready_package_v19_uiux_minfix.zip`

## 今回の修正方針

大幅改修ではなく、実機で出ていた3点を優先して最小修正しました。

1. 一覧を押しても、一覧が出たかどうかわからない
2. ランダムボタンの押下色が残って見える
3. ファーストビューでヘッダーが高く、超訳が見えにくい

## UIUXレビュー結果

### 1. 一覧表示

問題：スマホでは一覧がカードの下側に出る構造だと、ボタンを押しても画面内で変化が見えにくい。

対応：
- スマホ幅では一覧パネルをカードより上に表示
- 一覧ボタンを開閉状態で `一覧` / `一覧を閉じる` に変更
- `aria-expanded` を付与
- 一覧の見出しを `カード一覧（160件）` のように件数つきに変更
- 一覧表示時にトーストで `一覧を表示しました` を表示
- 一覧項目を押したらカード表示へ戻る

### 2. ランダムボタン

問題：押下後の色変化が残って見える。

対応：
- 永続的な `pressed` クラスを廃止
- 一瞬だけ発火する `tapFlash` アニメーションに変更
- 360ms後にクラス削除
- `blur()` を入れてフォーカス由来の見え残りを抑制
- 可能な場合は現在と同じカードを引かないように変更

### 3. ファーストビュー

問題：スマホでロゴ・説明・操作エリアが高く、超訳が見えにくい。

対応：
- スマホ幅でヘッダーを圧縮
- サブ説明はスマホでは非表示
- ジャンルボタンを横スクロール可能なチップに変更
- 検索・一覧まわりの高さを圧縮
- カード内の余白、本文、読み、タブ、超訳の余白を調整
- iPhone SE相当の 375×667 でも超訳本文がファーストビュー内に入ることを確認
